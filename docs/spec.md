# ネットワークシミュレータ 仕様書 v3.4

---

## 1. 概念定義

### ノード（5種類）

| 種別 | 役割 | ポート数 | 設定できる項目 |
|---|---|---|---|
| Router | 異なるネットワーク間の転送 | 8 | 論理IF・IPアドレス・スタティックルーティング |
| Switch | 同一ネットワーク内の転送 | 8 | IPアドレス・VLAN |
| PC | Ping送信元・Webブラウザ送信元 | 1 | IPアドレス・サブネットマスク・デフォルトGW・DNSサーバ |
| Server | Ping応答元・Webサーバ | 8 | IPアドレス・サブネットマスク・デフォルトGW・DNSサーバ・ホストドメイン |
| DNSサーバ | ドメイン名前解決 | 8 | IPアドレス・サブネットマスク・デフォルトGW・Aレコード |

### ポート

| 種別 | ポート名の形式 | 本数 |
|---|---|---|
| Router | GigabitEthernet0/0 〜 GigabitEthernet0/7 | 8 |
| Switch | GigabitEthernet0/0 〜 GigabitEthernet0/7 | 8 |
| PC | eth0 | 1 |
| Server | eth0 〜 eth7 | 8 |
| DNSサーバ | eth0 〜 eth7 | 8 |

### リンク
- 2つのノードの各ポートを指定して繋ぐ線
- 同じノード間に1本のみ存在可能（重複リンク禁止）
- 自己ループ禁止（同じノードのポート同士は繋げない）
- 1ポートに1リンクのみ接続可能

---

## 2. 制約

- 物理IFの装脱着は行わない（RouterのIFは論理IFのみ）
- 重複リンク禁止・自己ループ禁止・1ポートに1リンクのみ
- ノード削除時は紐づくリンクも同時に削除され、ポートが空きに戻る
- DNSのAレコードは1ドメインにつき1レコードのみ

## 3. 画面構成・遷移

### グローバルタブナビゲーション（全画面共通）

- 全ページの上部に共通のタブナビゲーションを表示する
- タブは左側にアプリタイトル、右側に3つのタブを配置する

| タブ名 | 遷移先 | 説明 |
|---|---|---|
| 概要 | /（トップページ） | サービス紹介・ランディングページ |
| 演習 | /exercises | 演習選択画面 |
| フリー | /simulation | 自由シミュレーション |

- 現在表示中のページに対応するタブはアクティブ状態でハイライトする
- タブナビゲーションは `components/shared/GlobalNav.tsx` として実装し
  `app/layout.tsx` で全ページに適用する

### 画面遷移図

```
グローバルタブ（全画面に常に表示）
  ├── 概要タブ  →  /（トップページ）
  ├── 演習タブ  →  /exercises（演習一覧）
  │     ├── 演習1カード  →  /exercises/1
  │     └── 演習2カード  →  /exercises/2
```

## 4. トップページ（/）

### コンセプト
- ネットワーク学習の初学者に向けたサービス紹介ページ
- 一般的なSaaSサービスのランディングページのようなモダンでおしゃれなデザイン

### 掲載内容

**ヒーローセクション（ファーストビュー）**
- キャッチコピー: 「パケットの動きを、目で学ぶ。」
- サブコピー:
  「ネットワーク初学者のための学習シミュレーター。
   実際の通信の流れをリアルタイムに可視化することで、
   教科書では伝わりにくいパケットの動きを直感的に理解できます。」
- CTAボタン: 「演習をはじめる」（/exercises へ遷移）
- CTAボタン: 「自由に試す」（/simulation へ遷移）

**特徴セクション（3カラム）**
- 🎯 ステップ実行
  「1ホップずつ通信を進めることで、パケットがどのノードを経由しているかを
   リアルタイムで確認できます。」
- 🔧 実機に近いConfig設定
  「Ciscoコンフィグ形式でルーターやスイッチの設定を入力できます。
   実機操作の感覚を学習段階から身につけられます。」
- 📚 演習問題で実力確認
  「VLANやルーティングなどのテーマ別演習で、
   知識を実践的なスキルに変換できます。」

**演習紹介セクション**
- 演習カードを2枚並べて表示（演習一覧画面と同じカードを流用）
- 「すべての演習を見る」ボタン（/exercises へ遷移）

### デザイン要件
- 背景: ダークテーマ（濃紺・黒系）をベースとする
- アクセントカラー: シアン・青緑系のグラデーション
- フォント: モダンでクリーンなサンセリフ体
- ヒーローセクションにはネットワークトポロジを想起させる
  幾何学的なアニメーション背景（SVGやCSSで実装）を配置する
- セクション間はなだらかなグラデーションで区切る
- 各要素はスクロールに合わせてフェードインするアニメーションを付ける

## 5. シミュレーションモード（/simulation）

自由にノードを配置・設定・通信できるモード。

### 5-1. GUI全体レイアウト

```
┌─────────────────────────────────────────────────────┐
│  [グローバルタブナビゲーション]                      │
├──────────┬──────────────────────────────┬───────────┤
│  ノード  │  キャンバス  [シミュトグル]  │ 設定パネル│
│ パレット │                              │           │
│  （左）  ├──────────────────────────────┤           │
│          │  操作バー・ログ              │           │
└──────────┴──────────────────────────────┴───────────┘
```

### 5-2. ノードアイコン

| 種別 | react-icons |
|---|---|
| Router | MdRouter |
| Switch | MdDeviceHub |
| PC | MdComputer |
| Server | MdStorage |
| DNSサーバ | MdPublic |

### 5-3. ノード操作

- **配置**: 左パレットからキャンバスへドラッグ＆ドロップ
- **移動**: キャンバス上のノードをドラッグ
- **初期名**: 種別ごとに自動採番（Router1, Router2 / Switch1 / PC1 / Server1 / DNS1）
- **設定を開く**: ノードを右クリック →「設定」→ 右パネルに表示
- **削除**: ノードを右クリック →「削除」
- **ホバー・選択時の表示**: ノードをホバー・クリックしても上下左右に
  接続点（●）を表示しない。接続点はノードをクリックしてIF選択ポップアップ
  から選ぶ方式のみとする。

### 5-4. リンク操作

**作成フロー（2ステップポート選択）:**
1. ノードをクリックすると空きポート一覧がポップアップで表示される
2. ポップアップからポートを選択すると接続元が確定し、仮線が表示される
3. 接続先ノードをクリックすると空きポート一覧がポップアップで表示される
4. 接続先ポートを選択するとリンクが作成される
5. ESCキーまたはキャンバス背景クリックでキャンセル

**選択・削除・再接続:**
- リンクをクリックすると選択状態になり黄色にハイライトされる
- 選択中にDeleteキーを押すとリンクが削除される
- リンクの端点ドラッグで切断・別ノードへ再接続モードに入る
- キャンバス背景クリックで選択解除
- リンクを右クリック →「削除」でも削除可能

### 5-5. ノード設定パネル（右側）

**Router設定パネル**
- ポート一覧（GigabitEthernet0/0〜0/7）・接続状態（空き/使用中）表示
- 接続中ポートのIPアドレス・サブネットマスク入力
- Ciscoコンフィグ入力エリア（CLI直打ち）
  - 対応コマンド: `interface` / `ip address` / `ip route` / `no shutdown`
- Ping送信ボタン（宛先IP入力欄つき）

**Switch設定パネル**
- ポート一覧（GigabitEthernet0/0〜0/7）・接続状態表示
- IPアドレス・サブネットマスク入力欄
- Ciscoコンフィグ入力エリア（CLI直打ち）
  - 対応コマンド: `interface` / `ip address` / `vlan` / `switchport`
- VLAN一覧表示

**PC設定パネル**
- eth0の接続状態表示
- IPアドレス・サブネットマスク・デフォルトGW・DNSサーバ入力欄
- Ping送信ボタン（宛先IP or ホスト名入力欄つき）
- Webアクセスボタン（URL入力欄つき）

**Server設定パネル**
- ポート一覧（eth0〜eth7）・接続状態表示
- IPアドレス・サブネットマスク・デフォルトGW・DNSサーバ入力欄
- ホストドメイン一覧（追加・削除可能）

**DNSサーバ設定パネル**
- ポート一覧（eth0〜eth7）・接続状態表示
- IPアドレス・サブネットマスク・デフォルトGW入力欄
- Aレコード一覧（ドメイン名・IPアドレスの追加・削除）

### 5-6. シミュレーションモードトグル

- キャンバス右上にシミュレーションモードのトグルボタンを配置する

**ONのとき（ステップ実行モード）:**
- スタートボタンを押すと通信が開始される
- 各ノード間ごとに一時停止し「次へ」ボタンが有効になる
- 「次へ」を押すまで現在ホップのアニメーションをループ再生する
- 往復通信を可視化する（outbound → return の順に全ホップ表示）

**OFFのとき（即時確認モード）:**
- 疎通確認ボタンを押すと即座に結果を表示する
- アニメーションは表示しない・結果のみログエリアに表示する

### 5-7. パケットアニメーション（シミュレーションモードON時のみ）

- SVG の radialGradient を使った光点がリンクに沿って移動する
- CSSアニメーション（duration: 600ms・ease-in-out）
- 「次へ」を押すまで現在ホップをループ再生する
- 光点通過後のリンクは一時的にハイライト（stroke変更）
- 到達ノードはグロー効果で一瞬ハイライト
- 往復: outbound全ホップ → return全ホップの順で表示

### 5-8. 操作バー（下部）

- スタート・次へ・リセットボタン
- ログエリア（各ホップの通信内容・現在ホップをハイライト）

### 5-9. トポロジの保存・読み込み

- **保存**: 上部メニューから「保存」→ JSONファイルとしてダウンロード
- **読み込み**: 上部メニューから「読み込み」→ JSONを選択して復元
- 読み込み前に上書き確認ダイアログを表示する

## 6. 演習モード

### 6-1. 演習一覧画面（/exercises）

- 演習カードをグリッド上に配置する
- 各カードに表示する要素: 難度バッジ・タイトル・演習内容の説明文
- カードをクリックすると各演習画面へ遷移する

### 6-2. 演習画面の共通仕様

**Answerボタン:**
- クリアの場合: 画面中央に「CLEAR!!」をオーバーレイ表示
  - 「演習一覧に戻る」ボタンを表示
- 失敗の場合: 「NG」と失敗理由をオーバーレイ表示
  - 「もう一度」ボタンでオーバーレイを閉じて再挑戦

### 6-3. 演習1: VLAN制御（/exercises/1）

**難度**: 初級 / **タイトル**: VLAN制御
**説明**: 4つのPCを一つのスイッチで2組のネットワークに分離する

**初期状態:**
- PC1・PC2・PC3・PC4（正方形の四隅）・Switch1（中央）を配置済み
- ノードの追加・削除・移動は不可
- 配線とConfig設定のみ可能

**正解判定:**
1. PC1とPC2が同一VLANに属しPing疎通が取れる
2. PC3とPC4が同一VLANに属しPing疎通が取れる
3. PC1/PC2とPC3/PC4の間でPing疎通が取れない

### 6-4. 演習2: ルーティング設定（/exercises/2）

**難度**: 初級 / **タイトル**: ルーティング設定
**説明**: 別セグメントの2台のPCをルーターで繋いで疎通を取る

**初期状態:**
- PC1（192.168.1.1/255.255.255.0 設定済み・変更不可）
- PC2（192.168.2.1/255.255.255.0 設定済み・変更不可）
- パレットにはSwitchとRouterのみ表示
- 追加上限: Router×1・Switch×2

**正解判定:**
1. PC1とPC2の間でPing疎通が取れる
2. 経路にルーターが含まれている

- 演習の進捗保存
