# ネットワークシミュレータ 仕様書 v3.2

---

## 1. 概念定義

### ノード（5種類）

| 種別 | 役割 | ポート数 | 設定できる項目 |
|---|---|---|---|
| Router | 異なるネットワーク間の転送 | 8 | 論理IF・IPアドレス・スタティックルーティング |
| Switch | 同一ネットワーク内の転送 | 8 | IPアドレス・VLAN |
| PC | Ping送信元・Webブラウザ送信元 | 1 | IPアドレス・サブネットマスク・デフォルトGW・DNSサーバ |
| Server | Ping応答元・Webサーバ | 8 | IPアドレス・サブネットマスク・デフォルトGW・DNSサーバ・ホストドメイン |
| DNSサーバ | ドメイン名前解決 | 8 | IPアドレス・サブネットマスク・デフォルトGW・Aレコード |

### ポート

| 種別 | ポート名の形式 | 本数 |
|---|---|---|
| Router | GigabitEthernet0/0 〜 GigabitEthernet0/7 | 8 |
| Switch | GigabitEthernet0/0 〜 GigabitEthernet0/7 | 8 |
| PC | eth0 | 1 |
| Server | eth0 〜 eth7 | 8 |
| DNSサーバ | eth0 〜 eth7 | 8 |

### リンク
- 2つのノードの各ポートを指定して繋ぐ線
- 同じノード間に1本のみ存在可能（重複リンク禁止）
- 1ポートに1リンクのみ接続可能

---

## 2. 制約

- 物理IFの装脱着は行わない（RouterのIFは論理IFのみ）
- 重複リンク禁止
- 1ポートに1リンクのみ
- ノード削除時は紐づくリンクも同時に削除され、ポートが空きに戻る
- DNSのAレコードは1ドメインにつき1レコードのみ

## 5. シミュレーションモード（/simulation）

自由にノードを配置・設定・通信できるモード。

### 5-1. GUI全体レイアウト

```
┌──────────┬────────────────────────────┬──────────────┐
│  ノード  │     キャンバス             │  設定パネル  │
│ パレット │                            │  （右側）    │
│  （左）  │                            │              │
│          ├────────────────────────────┤              │
│          │  シミュレーション操作バー  │              │
└──────────┴────────────────────────────┴──────────────┘
```

### 5-2. ノード操作
- **配置**: 左パレットからキャンバスへドラッグ＆ドロップ
- **移動**: キャンバス上のノードをドラッグ
- **初期名**: 種別ごとに自動採番（Router1, Router2 / Switch1 / PC1 / Server1 / DNS1）
- **設定を開く**: ノードを右クリック →「設定」→ 右パネルに表示
- **削除**: ノードを右クリック →「削除」

### 5-3. リンク操作
- **作成**:
  1. ノードにカーソルを近づけると接続点（ポート）が出現
  2. 接続元ポートをクリック → 接続先選択モードへ
  3. 接続先ノードをクリック → 空きポート一覧がドロップダウン表示
  4. 接続先ポートを選択 → リンク作成完了
- **削除**: リンクを右クリック →「削除」

### 5-4. ノード設定パネル（右側）

**Router設定パネル**
- ポート一覧（GigabitEthernet0/0〜0/7）・接続状態表示
- 接続中ポートのIPアドレス・サブネットマスク入力
- Ciscoコンフィグ入力エリア（CLI直打ち）
  - 対応コマンド: `interface` / `ip address` / `ip route` / `no shutdown`
- Ping送信ボタン（宛先IP入力欄つき）

**Switch設定パネル**
- ポート一覧（GigabitEthernet0/0〜0/7）・接続状態表示
- IPアドレス・サブネットマスク入力欄
- Ciscoコンフィグ入力エリア（CLI直打ち）
  - 対応コマンド: `interface` / `ip address` / `vlan` / `switchport`
- VLAN一覧表示

**PC設定パネル**
- eth0の接続状態表示
- IPアドレス・サブネットマスク・デフォルトGW・DNSサーバ入力欄
- Ping送信ボタン（宛先IP or ホスト名入力欄つき）
- Webアクセスボタン（URL入力欄つき）

**Server設定パネル**
- ポート一覧（eth0〜eth7）・接続状態表示
- IPアドレス・サブネットマスク・デフォルトGW・DNSサーバ入力欄
- ホストドメイン一覧（追加・削除可能）

**DNSサーバ設定パネル**
- ポート一覧（eth0〜eth7）・接続状態表示
- IPアドレス・サブネットマスク・デフォルトGW入力欄
- Aレコード一覧（ドメイン名・IPアドレスの追加・削除）

### 5-5. Ping通信とパケット可視化
- PC・Serverの設定パネルから発生
- 疎通判定: IPアドレスとルーティング設定をもとに判定
- 失敗メッセージはWindowsコマンドプロンプトのping表示に準拠
- **パケット可視化**: 「次へ」ボタンでパケットアイコンがリンク上を移動するアニメーション
  - アクティブなリンクは色・太さが変化
  - 到達ノードはハイライト表示
  - シミュレーションログに各ホップの説明を表示

### 5-6. Webアクセス通信
- PCの設定パネルからURL入力で発生
- 通信フロー: DNS名前解決 → HTTPアクセス
- 結果はブラウザ風UI（アドレスバーにURL・結果表示）

### 5-7. シミュレーション操作バー（下部）
- スタートボタン・次へボタン・リセットボタン
- ログエリア（各ホップの通信内容をテキスト表示）

### 5-8. トポロジの保存・読み込み（セーブポイント機能）
- **保存**: 上部メニューから「保存」→ JSONファイルとしてダウンロード
- **読み込み**: 上部メニューから「読み込み」→ JSONファイルを選択して復元
- 読み込み前に上書き確認ダイアログを表示
- 保存対象: ノード一覧・ポート状態・リンク一覧・設定値・座標

## 6. 演習モード

### 6-1. 演習一覧画面（/exercises）

- 演習カードをグリッド上に配置する
- 各カードには以下の3要素を表示する
  - **難度バッジ**（例: 初級・中級・上級）
  - **タイトル**（例: VLAN制御）
  - **演習内容の説明文**（例: 4つのPCを一つのスイッチで2組のネットワークに分離する）
- カードをクリックすると各演習画面へ遷移する
- 画面上部に「← トップに戻る」リンクを表示する

### 6-2. 演習画面の共通仕様

**レイアウト**
```
┌─────────────────────────────────────────────────┐
│  演習タイトル・説明                              │
├──────────┬──────────────────────────┬───────────┤
│  パレット │     キャンバス           │ 設定パネル│
│（制限あり）│                         │           │
│          ├──────────────────────────┤           │
│          │  操作バー・Answerボタン  │           │
└──────────┴──────────────────────────┴───────────┘
```

**Answerボタン**
- 画面下部の操作バーに配置する
- クリックすると演習ごとの正解判定ロジックを実行する
- **クリア時**: 画面中央に「CLEAR!!」を大きく表示する（オーバーレイ）
  - 「演習一覧に戻る」ボタンを表示する
- **失敗時**: 画面中央に「NG」と失敗理由を表示する（オーバーレイ）
  - 「もう一度」ボタンでオーバーレイを閉じて再挑戦できる

### 6-3. 演習1: VLAN制御（/exercises/1）

**難度**: 初級
**タイトル**: VLAN制御
**説明**: 4つのPCを一つのスイッチで2組のネットワークに分離する

**初期状態**
- キャンバスにあらかじめ以下のノードが配置されている（位置は固定）
  - PC1・PC2・PC3・PC4（正方形の四隅に配置）
  - Switch1（中央に配置）
- 各ノードのIPアドレスは未設定（学習者が設定する）
- ノードの追加・削除・移動は不可

**操作の制限**
- パレットは表示しない（ノード追加不可）
- ノードのドラッグ移動は不可
- 削除メニューを表示しない
- 配線（リンク作成）は可能
- Switch1のConfig設定は可能

**正解判定（Answerボタン押下時）**
- 以下の条件をすべて満たす場合にクリアとする
  1. PC1とPC2が同一VLANに属し、Ping疎通が取れる
  2. PC3とPC4が同一VLANに属し、Ping疎通が取れる
  3. PC1またはPC2とPC3またはPC4の間でPing疎通が取れない（VLAN分離できている）
- 失敗時は満たしていない条件をメッセージで表示する

### 6-4. 演習2: ルーティング設定（/exercises/2）

**難度**: 初級
**タイトル**: ルーティング設定
**説明**: 別セグメントの2台のPCをルーターで繋いで疎通を取る

**初期状態**
- キャンバスにあらかじめ以下のノードが配置されている（位置は固定）
  - PC1（IPアドレス: 192.168.1.1 / 255.255.255.0 設定済み・変更不可）
  - PC2（IPアドレス: 192.168.2.1 / 255.255.255.0 設定済み・変更不可）
- PC1・PC2のIPアドレス入力欄はグレーアウトして編集不可

**操作の制限**
- パレットにはSwitchとRouterのみ表示（PC・Server・DNSは追加不可）
- 配置できるノードの上限: Router×1・Switch×2
- ノードの移動は可能
- 配置済みのPC1・PC2の削除は不可
- 追加したSwitch・Routerの削除は可能

**正解判定（Answerボタン押下時）**
- 以下の条件をすべて満たす場合にクリアとする
  1. PC1とPC2の間でPing疎通が取れる
  2. ルーターが経路に含まれている（直結は不正解）
- 失敗時は満たしていない条件をメッセージで表示する

## 7. スコープ外（未対応）

- OSPFなどのダイナミックルーティング
- NATアドレス変換
- ACL（アクセスコントロールリスト）
- IPv6
- マルチキャスト通信
- 演習の進捗保存（ページをリロードするとリセットされる）
