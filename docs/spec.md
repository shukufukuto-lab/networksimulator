# ネットワークシミュレータ 仕様書 v3.0（確定版）

---

## 1. 概念定義

### ノード（5種類）

| 種別 | 役割 | 設定できる項目 |
|---|---|---|
| Router | 異なるネットワーク間の転送 | 論理IF・IPアドレス・スタティックルーティング |
| Switch | 同一ネットワーク内の転送 | IPアドレス・VLAN |
| PC | Ping送信元・Webブラウザ送信元 | IPアドレス・サブネットマスク・デフォルトGW・DNSサーバ |
| Server | Ping応答元・Webサーバ | IPアドレス・サブネットマスク・デフォルトGW・DNSサーバ・ホストドメイン |
| DNSサーバ | ドメイン名前解決 | IPアドレス・サブネットマスク・デフォルトGW・Aレコード |

### リンク
- 2つのノードを繋ぐ線
- 同じノード間には1本のみ存在可能（重複リンク禁止）

---

## 2. 制約

- 物理IFの装脱着は行わない（RouterのIFは論理IFのみ）
- 重複リンク禁止（同じノード間に2本以上の線は作れない）
- ノード削除時は紐づくリンクも同時に削除される
- DNSのAレコードは1ドメインにつき1レコードのみ（重複ドメイン禁止）

## 3. 振る舞い

### 3-1. GUI全体レイアウト

```
┌──────────┬────────────────────────────┬──────────────┐
│          │                            │              │
│  ノード  │     キャンバス             │  設定パネル  │
│ パレット │  （メイン作業エリア）      │  （右側）    │
│  （左）  │                            │              │
│          ├────────────────────────────┤              │
│          │  シミュレーション操作バー  │              │
└──────────┴────────────────────────────┴──────────────┘
```

### 3-2. ノード操作

- **配置**: 左パレットからキャンバスへドラッグ＆ドロップ
- **移動**: キャンバス上のノードをドラッグ
- **初期名**: 種別ごとに自動採番（Router1, Router2 / Switch1 / PC1 / Server1 / DNS1）
- **設定を開く**: ノードを右クリック →「設定」を選択 → 右パネルに表示
- **削除**: ノードを右クリック →「削除」を選択（紐づくリンクも同時削除）

### 3-3. リンク操作

- **作成**: ノードにカーソルを近づけると接続点（●）が出現 → その接続点からドラッグして別ノードへ繋ぐ
- **削除**: リンクを右クリック →「削除」を選択

### 3-4. ノード設定パネル（右側）

ノードを右クリック →「設定」で右パネルに表示される。

**Router設定パネル**
- 論理IF一覧（GigabitEthernet0/0 から自動採番、ユーザー入力不要）
- 各IFのIPアドレス・サブネットマスク表示
- Ciscoコンフィグ入力エリア（CLIテキスト直打ち）
  - 対応コマンド: `interface`, `ip address`, `ip route`, `no shutdown`
- Ping送信ボタン（宛先IP入力欄つき）

**Switch設定パネル**
- IPアドレス・サブネットマスク入力欄
- Ciscoコンフィグ入力エリア（CLIテキスト直打ち）
  - 対応コマンド: `interface`, `ip address`, `vlan`, `switchport`
- VLAN一覧表示

**PC設定パネル**
- IPアドレス・サブネットマスク・デフォルトGW・DNSサーバ入力欄
- Ping送信ボタン（宛先IP or ホスト名入力欄つき）
- Webアクセスボタン（URL入力欄つき）

**Server設定パネル**
- IPアドレス・サブネットマスク・デフォルトGW・DNSサーバ入力欄
- ホストドメイン一覧（追加・削除可能）

**DNSサーバ設定パネル**
- IPアドレス・サブネットマスク・デフォルトGW入力欄
- Aレコード一覧（ドメイン名・IPアドレスの追加・削除）

### 3-5. Ping通信

- PC・Serverの設定パネルから発生
- 疎通判定: IPアドレスとルーティング設定をもとに判定
- 失敗メッセージはWindowsコマンドプロンプトのping表示に準拠
  - `要求がタイムアウトしました。`
  - `宛先ホストに到達できません。`
  - `TTLが転送中に期限切れになりました。`
  - `一般的な失敗です。`

### 3-6. Webアクセス通信

- PCの設定パネルからURL入力で発生
- 通信フロー: DNS名前解決 → HTTPアクセス
- 名前解決失敗・経路なし・Server未応答の場合はエラー表示
- 結果はシンプルなブラウザ風UI（アドレスバーにURL・結果表示）

### 3-7. シミュレーション実行

- 画面下部にシミュレーション操作バーを配置
- **スタートボタン**: 押すと通信がステップ実行モードに入る
- **次へボタン**: 1ホップずつ通信が進み、経路がGUI上でアニメーション表示される
- **リセットボタン**: シミュレーションを初期状態に戻す

### 3-8. トポロジの保存・読み込み

- **保存**: 画面上部メニューから「保存」→ JSONファイルとしてダウンロード
- **読み込み**: 画面上部メニューから「読み込み」→ JSONファイルを選択してトポロジを復元
- 保存対象: ノード一覧・リンク一覧・各ノードの設定値・ノードの座標

## 4. スコープ外（未対応）

- NATアドレス変換
- ACL（アクセスコントロールリスト）
- IPv6
- マルチキャスト通信
